# 智能文件处理工具 GUI 使用指南

## 🚀 快速开始

### 启动方式

**方法一：双击启动（推荐）**
- Windows: 双击 `start.bat`
- Linux/Mac: 运行 `bash start.sh`

**方法二：命令行启动**
```bash
python run_gui.py
```

**方法三：直接运行**
```bash
python improved_gui.py
```

## 🎯 界面概览

GUI采用现代化选项卡设计，包含以下4个主要功能区：

### 📁 文件处理

**单文件处理：**
1. 点击"选择文件"按钮选择要处理的文件
2. 点击"选择输出"设置保存位置
3. 选择输出格式：
   - **摘要**：包含统计信息的简洁报告
   - **JSON**：详细的结构化数据
   - **纯文本**：仅处理后的文本内容
4. 勾选需要的处理选项：
   - ✅ 情感分析：分析文本情感倾向
   - ✅ 实体识别：提取人名、组织名等
   - ✅ 数字提取：自动识别数字
   - ✅ 日期提取：识别各种日期格式
5. 点击"🚀 开始处理"

**批量处理：**
1. 点击"选择文件夹"选择包含多个文件的目录
2. 设置输出文件夹
3. 其他步骤同单文件处理

### ⚙️ 配置

**查看和修改配置：**
- 配置以JSON格式显示，可直接编辑
- 主要配置项：
  - `max_file_size_mb`: 最大文件大小限制
  - `max_workers`: 并发处理线程数
  - `supported_formats`: 支持的文件格式
  - `detect_language`: 是否启用语言检测
  - `sentiment_analysis`: 是否启用情感分析

**配置操作：**
- 🔄 **刷新配置**：重新加载配置文件
- 🔧 **重置为默认**：恢复出厂设置
- 💾 **保存配置**：保存当前修改

### 📊 结果

**处理统计：**
- 语言检测结果
- 文本统计信息（字符数、词数、句子数）
- 提取的数字和日期数量
- 识别的实体数量

**详细结果：**
- 完整的JSON格式处理结果
- 包含所有分析数据

**结果操作：**
- 📋 **复制结果**：复制到剪贴板
- 💾 **导出结果**：保存为文件
- 🧹 **清空结果**：清除当前结果

- 📖 **使用说明**：详细的功能介绍
- 🔧 **技术特性**：技术规格说明
- ℹ️ **关于**：版本和开发信息

## 🎨 特色功能

### 💾 多格式输出
- **摘要格式**：适合快速查看
- **JSON格式**：便于程序处理

### ⚡ 高效处理
- **并发处理**：支持多线程批量处理
- **内存优化**：大文件分块处理
- **格式广泛**：支持 txt、csv、json、pdf、xlsx 等

## 📝 使用技巧

### 🎯 最佳实践

1. **文件选择**
   - 支持的格式：`.txt`, `.csv`, `.json`, `.pdf`, `.xlsx`
   - 建议文件大小不超过100MB（可在配置中调整）

2. **输出格式选择**
   - 快速查看 → 选择"摘要"
   - 程序集成 → 选择"JSON"
   - 文本编辑 → 选择"纯文本"

3. **批量处理**
   - 确保输入文件夹只包含需要处理的文件
   - 输出文件夹会自动创建
   - 处理完成后在输出文件夹查看结果

4. **配置优化**
   - 根据计算机性能调整`max_workers`（线程数）
   - 根据需要调整`max_file_size_mb`（文件大小限制）

### ⚠️ 注意事项

1. **首次使用**
   - 确保已安装所需的Python包：`pip install -r requirements.txt`
   - 下载spaCy模型：`python -m spacy download en_core_web_sm zh_core_web_sm`

2. **大文件处理**
   - PDF文件处理可能较慢，请耐心等待
   - 超大文件建议分割后处理

3. **网络要求**
   - 首次运行可能需要下载NLTK数据
   - 确保网络连接正常

## 🔧 故障排除

### 常见问题

**Q: GUI无法启动**
- A: 检查Python版本（需要3.7+）和tkinter安装
- A: 运行 `python test_gui.py` 进行诊断

**Q: 处理速度慢**
- A: 在配置中增加`max_workers`数值
- A: 确保有足够的内存空间

**Q: 某些文件无法处理**
- A: 检查文件格式是否在支持列表中
- A: 确认文件没有损坏且可以正常打开

**Q: 结果不准确**
- A: 某些功能需要特定的spaCy模型
- A: 运行 `python -m spacy download <model_name>` 下载模型

### 获取帮助

1. 查看错误日志：`app.log`
2. 运行诊断：`python test_improvements.py`
3. 查看详细文档：`README_improved.md`
